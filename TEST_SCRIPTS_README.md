# 資訊安全專案測試腳本使用說明

## 執行檔說明

### `run-tests.bat` - 主要測試套件
- 互動式選單選擇測試類型
- 自動執行對應測試腳本

### `test-local.bat` - 本地測試
- 安裝依賴
- 啟動後端和前端服務
- 提供測試指引
- 測試完成後自動清理

### `test-docker.bat` - Docker 測試
- 檢查 Docker 環境
- 建置和運行容器
- 自動清理測試環境

## 使用方法

### 快速測試
```bash
# 雙擊 run-tests.bat 並選擇測試類型
run-tests.bat
```

### 直接運行
```bash
# 本地測試
test-local.bat

# Docker 測試
test-docker.bat
```

## 腳本功能

### 本地測試腳本特性
- ✅ 自動依賴安裝
- ✅ 並行啟動服務
- ✅ 測試指引顯示
- ✅ 服務清理
- ✅ 錯誤處理

### Docker 測試腳本特性
- ✅ 環境檢查
- ✅ 容器建置
- ✅ 自動清理
- ✅ 錯誤處理

## 注意事項

1. **權限**: 確保有管理員權限運行 Docker 相關命令
2. **端口**: 確保 3000 和 3001 端口未被占用
3. **清理**: 測試後腳本會自動清理，但可手動檢查
4. **日誌**: 如需詳細日誌，可在運行時查看控制台輸出

## 故障排除

### 本地測試失敗
- 檢查 Node.js 和 npm 已安裝
- 確認端口 3000/3001 未被占用
- 手動刪除 node_modules 重新安裝

### Docker 測試失敗
- 確保 Docker Desktop 運行
- 檢查 Docker 資源分配充足
- 清除 Docker 快取: `docker system prune`

### 服務啟動問題
- 等待服務完全啟動 (腳本已包含延遲)
- 檢查防火牆設定
- 查看詳細錯誤日誌
